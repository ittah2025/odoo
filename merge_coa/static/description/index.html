  2. System Requirements
    1. This action can only be performed by users with the Accounting – Invoicing – Chief Accountant role.
    2. You can merge transactions from any number of accounts that you wish to consolidate.
    3. You can only merge accounts or transfer transactions between accounts of the same type.
    4. Transaction transfers can only be performed for transactions within open lock dates.
    5. If you wish to transfer transactions that fall on the lock date, you must temporarily disable the lock date to carry out the transfer process. You can re-enable the lock date after the process is completed.
    6. For transactions that need to be transferred after the lock date, you can specify a date range for the start and end dates.
    7. You can perform a rollback of transactions on the source account, following the batch order starting from the most recent action.
  3. Usage Steps
    1. Preparation Steps
      1. Installation
        • Ensure you have the accounting module installed.
        • Install the Merge Account & Transfer Account Transaction module.
      2. Data Backup

        Every successful endeavor begins with thorough preparation. Perform a data backup to ensure all your important information remains safe.

      3. Account Identification

        Take some time to analyze your accounts. Determine which accounts need to be merged and ensure you understand the transactions within them. Also, confirm that the types of the accounts to be merged are the same.

      4. Select Target Account

        Identify the account that will serve as the target for merging. Ensure this account is ready to receive transactions and has the same account type as the source accounts.

    2. Step for Use
      1. Go to the Chart of Accounts menu.
      2. Customize the accounts in the Chart of Accounts According to Your Current Needs.

        4. Rollback Function

          If you make an error in selecting the accounts to merge or transfer, don’t worry; we have provided a rollback feature. This allows you to easily and quickly restore transactions to their original accounts.

          • Using the previous case, you may wish to revert the transaction for account 11110010 Petty Cash, as it was mistakenly included in the merge.

          • Go to the Chart of Accounts page, then select "view" for the account from which you want to restore transactions.


          • View the merge history tab to see the history of transactions transferred via the "Merge Account & Transfer Account Transaction" module. Check the batch column in the merge history tab, to determine the order in which transactions for the relevant account were moved. You can perform a rollback on transactions moved as part of the merge or transfer, according to the most recent batch order. Click the Rollback Merge Account button to initiate the rollback feature.


          • Result of the Rollback Merge Account process: Transactions for the account utilizing the rollback feature will be restored to their original state.
